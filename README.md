## doc-coauthoring
`doc-coauthoring` は、提案書・設計書・仕様書などのドキュメントを、AIと一緒に段階的に作るためのSKILLです。

流れはシンプルで、以下の3段階です。

1. 背景や目的、読者を整理する  
2. セクションごとに内容を詰めて文章化する  
3. 読者目線で分かりにくい点がないか確認する  

要するに、**いきなり全文を書くのではなく、情報整理 → 構成作成 → 執筆 → 見直し** の流れで、読みやすいドキュメントを作るためのSKILLです。


## frontend-design（Anthropic公式）

- **目的**  
  ありきたりではない、印象に残る高品質なフロントエンドを作るためのSkill

- **やること**  
  画面の目的・対象ユーザー・デザインの方向性を最初に決める

- **重要な考え方**  
  無難なAIっぽいUIではなく、はっきりした世界観を持たせる

- **重視する点**  
  タイポグラフィ、配色、アニメーション、レイアウト、背景や装飾

- **避けるもの**  
  量産型UI、ありがちな配色、無個性なフォント、テンプレっぽい見た目

- **ゴール**  
  デザイン性が高く、実際に動く本番品質のUIを作る

# skill-creator

`skill-creator` は、**Claude 用の Skill を作るための Skill** です。

## ひとことで言うと

- Skill の作り方をガイドしてくれる
- どんな構成にするか整理してくれる
- 再利用しやすい Skill にまとめるのを助けてくれる

## 何に使うか

たとえば、こんなときに使います。

- 毎回同じ手順を Claude にやらせたい
- 特定の業務に強い専用 Skill を作りたい
- チームで使える形に整理したい
- Skill の説明文や構成を整えたい

## イメージ

普通の Skill は「作業をするもの」ですが、  
`skill-creator` は **その Skill 自体を設計・作成するための補助役** です。

## ざっくり役割

- Skill の目的を明確にする
- 使う場面を整理する
- `SKILL.md` などの中身を考える
- わかりやすく再利用可能な形にまとめる

## つまり

`skill-creator` は、  
**Claude のための専用機能を、自分で作れるようにするためのメタSkill** です。
